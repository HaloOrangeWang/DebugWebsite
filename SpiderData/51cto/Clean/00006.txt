问题描述

今天小编在Python语言中，写了几行简单的代码，确定没有明显的语法错误，但运行时出现报错IndentationError:unexpected idenx。

Python中比较常见的缩进错误,尤其是对于初学者经常因为缩进不对出现报错而烦恼。

解决方案

Python是一种对缩进非常敏感的语言，无论是该缩进的地方没有缩进，还是不该缩进的地方缩进了，都会导致Python的解释器无法正常理解程序。常见的情况是Tab和空格的混用会导致错误或者缩进不对，而这是无法直接肉眼辨别的。

因此当出现报错时，显然应该仔细查看代码中的缩进是否规范，在检查Tab和空格时都要注意，可以用空格键替换Tab键输入。还有要注意的就是 : 后要进行缩进，例如if语句后就该缩进。同时也要注意代码对齐，例如下面for前就不该缩进。

def f(n):

if n == 5:

return 5

结语

通过运行上面的代码就可以明显感受到Python中缩进重要性了，对于初学者来说，早期就要注意到这点，不然在后期学习中出现这类报错容易打击学习积极性。所以写代码时就要注意是否需要缩进。
